# 家路通微信小程序 MVP

这是一个微信小程序 MVP。固定锚点路线保存在 `data/routes.js`，可选后端用于素材上传和路线配置同步。

## 家属自动创建路线

进入 `首页 → 家属配置 → 新建路线`，家属可以自定义路线名称，搜索起点和终点，选择步行或公交优先，自动生成决策锚点草稿，再逐步审核并发布。

创建路线时选择的是首页按钮位置，例如“首页按钮 1 / 首页按钮 2”。首页最终展示的按钮文字来自家属填写的路线名称，不再固定为“去妈妈家 / 回家”。

地点搜索、当前位置路线规划与步行明细展开，统一由家路通服务端代理请求百度地图。百度地图 AK 只配置在服务端，不放入小程序代码包。小程序需要在 `config/upload.js` 配置 `apiBaseUrl` 和 `uploadUrl`；家属端通过微信快速登录后获得操作权限。服务端需要配置：

```bash
JIALUTONG_BAIDU_MAP_KEY=百度地图服务端AK
JIALUTONG_WECHAT_APPID=微信小程序AppID
JIALUTONG_WECHAT_SECRET=微信小程序AppSecret
```

当前自动创建的路线保存为独立路线草稿。审核发布后，首页会优先读取对应按钮的已发布路线；未发布时继续使用原有固定路线。

当前固定路线主干为：

富友嘉园一期 → 步行或骑行至临洮路站 → 14 号线 → 曹杨路换乘 3 号线 → 江湾镇站 → 887 路 → 步行至彩虹湾墨翠里。

步行和骑行段必须将每一个转弯、过红绿灯、进出站动作拆成独立步骤。

## 低识字家人模式

- 步骤进入后自动播放本地真人语音，并每 30 秒重复一次。
- 接近目标点后提示家人查看照片，由家人确认“我找到了”后进入下一步。
- 持续远离目标点时震动并显示“请停下，不要继续走”。
- 求助界面向路人展示目的地、当前步骤和当前位置。
- 路线页只保留“再听一遍”“我找到了，下一步”和长按求助。
- 图片可配合 `direction` 显示醒目的方向箭头。

## 路线图片和真人语音

当前有 23 个步骤：去程 13 步、回程 10 步。

- 最稳妥配置：23 张步骤照片、23 段真人语音。
- 首页进入“家属配置路线素材”，可逐步拍摄照片、录音、试听和删除。
- 配置 `uploadUrl` 后，照片和语音会上传到家路通服务端，并同步到路线配置。
- 未配置上传服务时才使用 `wx.saveFile` 保存在当前手机；删除小程序、清理数据或更换手机后，本地素材可能丢失。

### 可选上传接口

在 `config/upload.js` 中填写 `uploadUrl` 后，家属配置页会改用 `wx.uploadFile` 上传素材。

- 请求方式：`multipart/form-data`
- 文件字段：`file`
- 附加字段：`routeId`、`stepNo`、`kind`
- `kind` 值：`image` 或 `audio`
- 接口响应：`{ "url": "https://example.com/files/xxx.jpg" }`

微信小程序后台需要将上传服务器域名加入 uploadFile 合法域名。

本地开发者工具使用 `http://127.0.0.1:8090` 时，需要关闭“校验合法域名”。真机中的 `127.0.0.1` 指向手机自身，必须改为 HTTPS 公网服务地址。
生产环境不要在小程序代码包中填写长期 `uploadToken`。家属进入“家属配置”后使用微信一键登录，上传时会自动携带服务端签发的登录 Token。小程序端不会再把 `uploadToken` 作为未登录时的鉴权兜底。

生产环境的服务域名固定在 `config/upload.js`。`config/upload.local.js` 仍可在本地开发或脚本验证时配置应急管理 Token；该文件已被 Git 忽略，不得提交到仓库。正式小程序应优先使用家属账号登录态。

仓库内已提供独立的家路通服务：`../jialutong-server`。

部署后配置示例：

```js
module.exports = {
  uploadUrl: "https://your-domain.example.com/api/files",
  apiBaseUrl: "https://your-domain.example.com",
  uploadToken: "仅供脚本或本地应急使用，生产小程序不要填写长期 Token"
};
```

图片、语音、动作说明、方向和坐标都会同步到服务端。家人更换手机后，首页会自动拉取远程路线配置。

## 安全限制

- 只有步行或骑行步骤的准确坐标和实地确认是启动路线的强制条件；照片、真人语音和方向箭头属于强烈建议配置项。
- 位置状态仅在后台辅助判断，不向家人展示技术信息。
- 小程序进入后台时会暂停轮询；返回后必须重新确认当前步骤。
- 行程进度保存在本机，重新进入路线时会恢复。
- 无后端版本无法自动通知家属或让家属远程查看位置，偏航时会停止路线并展示拨号求助界面。
- 本地模式会压缩照片，但 23 张照片和 23 段语音仍可能达到微信本地文件容量上限；正式使用应配置上传服务。
- 小程序内置的固定上传 Token 不能作为高安全鉴权方案；正式产品应由后端签发短期上传凭证。
- 家属端已经支持微信快速登录；首次登录会自动创建家庭、家属账号、默认老人档案和家属-老人绑定关系。路线创建、审核、上传、AI 和发布操作需要登录，老人端读取已发布路线不强制登录。

## 本地运行

1. 打开微信开发者工具。
2. 选择“导入项目”，目录选择本文件夹。
3. 使用测试号或自己的小程序 AppID。
4. 在开发者工具和真机上允许定位权限。

## 百度地图接入

- 小程序不直接请求百度地图，避免在代码包中暴露服务端 AK。
- 小程序通过家路通服务端执行地点搜索、步行规划和公交规划。
- 公交路线中的普通步行接驳段，会再次请求步行规划并提取转弯、过马路和最终接驳点。
- 站内换乘通道识别为 `TRANSFER`，不会生成错误的中途出站步骤。

## 家人路线模式

- 路线页不显示地图、坐标、定位精度等技术信息。
- 顶部固定显示目的地、剩余距离和当前进度。
- 大照片和真人语音是主要引导方式，任务口令不超过 8 个字。
- 定位仅在后台用于计算与当前锚点的距离和偏航提醒，不会自动切换步骤。
- 求助按钮固定在底部操作栏，长按 3 秒后联系家属。

## 上线前必须替换

- 家属必须陪同实地完整走一次路线。
- 在 `data/routes.js` 中为每个步行或骑行转弯点填写明确的左转/右转提示和准确坐标。
- 为每个转弯点拍摄面向行进方向的照片，并填写对应 `image`。
- 为每个步骤录制真人语音并填写对应 `audio`。
- 在家属配置页确认求助联系人姓名、与老人的关系和真实手机号。
- 确认临洮路站入口、曹杨路换乘方向、江湾镇站出口、887 路上下车站。
- 将地点照片放入例如 `assets/images/`，并填写每一步的 `image` 路径。
- 将音频放入例如 `assets/audio/`，并填写每一步的 `audio` 路径。
- 求助联系人电话没有默认值，发布前必须由家属在配置页填写真实联系人信息，不得使用测试号码。

定位每 5 秒刷新一次。接近当前步骤锚点时，页面提示家人查看照片，并等待家人手动确认进入下一步。
